Part 1 - Language Switcher (i18n)
What is i18n?
Internationalisation (i18n = 18 letters between 'i' and 'n') means writing your app so user-visible strings can be swapped for different languages at runtime without changing logic.

Standard Pattern:

Replace hardcoded strings with translation keys ('app.title', 'todo.add', etc.)
Provide one catalog file per language mapping keys to translations
At runtime, read user's preferred language from session and look up keys in the right catalog
The app uses Symfony Translation for steps 2-3. Template calls like {{ trans('some.key') }} go through the Translator object.

PHP Sessions Reminder
Sessions carry three pieces of state:

$_SESSION['lang'] - Active UI language ('en' or 'fr')
$_SESSION['totp_secret'] - TOTP secret during login setup (Parts 2 & 3)
$_SESSION['authenticated'] - Whether user verified TOTP (Part 3)
Important: session_start() must be called before any output and before $_SESSION is accessed.

Uncomment session_start(); near the top of index.php.

Part 2 - OTP Service (TOTP)
What is TOTP?
TOTP (Time-based One-Time Password, RFC 6238) is the algorithm behind Google Authenticator and Authy. Both server and app independently compute the same 6-digit code from:

A secret - random Base32 string exchanged during setup
The current time - rounded to 30-second windows
Both sides use same inputs + algorithm → same result without network exchange.

Setup Flow:


Server generates secret → renders QR code → user scans with authenticator app
        ↓                                              ↓
   stores secret in session              app shows new 6-digit code every 30s
        ↓
   user submits code → server verifies → authenticated
QR codes are rendered as inline SVG data URIs using bacon/bacon-qr-code - no external services needed.

Step 1 - Implement OtpService::generate()
In src/Services/OtpService.php:


public function generate(string $label): string
{
    $secret = $this->tfa->createSecret();                    // generate new Base32 secret
    $_SESSION['totp_secret'] = $secret;                     // store in session
    
    return $this->tfa->getQRCodeImageAsDataUri($label, $secret); // return QR code data URI
}
Step 2 - Implement OtpService::verify()

public function verify(string $input): bool
{
    $secret = $_SESSION['totp_secret'] ?? null;
    
    if (is_null($secret)) {
        return false;  // guard: no secret in session
    }
    
    return $this->tfa->verifyCode($secret, $input);  // verify submitted code
}

Part 3 - Auth Flow
Overview
Full authentication flow:


GET  /auth          → show username form
POST /auth/request  → generate OTP, display QR code
GET  /auth/verify   → show OTP entry form
POST /auth/verify   → check OTP → redirect to /todos or show error
POST /auth/logout   → destroy session → redirect to /auth
/todos routes are protected by AuthMiddleware checking $_SESSION['authenticated'].

Step 1 - Implement AuthMiddleware::__invoke()
In src/Middleware/AuthMiddleware.php:


public function __invoke(Request $request, RequestHandler $handler): Response
{
    if (!isset($_SESSION['authenticated']) || $_SESSION['authenticated'] !== true) {
        return $this->responseFactory->createResponse(302)
            ->withHeader('Location', $this->basePath . '/auth');
    }
    
    return $handler->handle($request);
}
Step 2 - Implement AuthController::requestOtp()
In src/Controllers/AuthController.php:


public function requestOtp(Request $request, Response $response): Response
{
    $data = (array) $request->getParsedBody();
    $username = trim($data['username'] ?? '');
    
    if (empty($username)) {
        return $response->withHeader('Location', $this->basePath . '/auth')->withStatus(302);
    }
    
    $_SESSION['username'] = $username;
    $qrCodeUri = $this->otpService->generate($username);
    
    $html = $this->twig->render('auth.html.twig', [
        'step' => 'otp_display',
        'qr_code' => $qrCodeUri,
        'base_path' => $this->basePath,
        'app_lang' => $_SESSION['lang'] ?? 'en',
    ]);
    $response->getBody()->write($html);
    return $response;
}
Step 3 - Implement AuthController::verifyOtp()

public function verifyOtp(Request $request, Response $response): Response
{
    $data = (array) $request->getParsedBody();
    $code = trim($data['code'] ?? '');
    
    if ($this->otpService->verify($code)) {
        $this->otpService->invalidate();
        $_SESSION['authenticated'] = true;
        return $response->withHeader('Location', $this->basePath . '/todos')->withStatus(302);
    } else {
        $html = $this->twig->render('auth.html.twig', [
            'step' => 'verify',
            'error' => 'auth.error_invalid',
            'base_path' => $this->basePath,
            'app_lang' => $_SESSION['lang'] ?? 'en',
        ]);
        $response->getBody()->write($html);
        return $response;
    }
}

Reflection - Request Flow
For an authenticated /todos request:


Request
  → SecurityHeadersMiddleware   (add headers on response)
  → MaintenanceMiddleware        (503 if flag present)
  → LoggerMiddleware             (record timing on response)
  → AuthMiddleware               (check authentication)
  → TodoController::index()      (build HTML)
Response ←─────────────────────────────────────
For an unauthenticated /todos request:


Request
  → SecurityHeadersMiddleware
  → MaintenanceMiddleware
  → LoggerMiddleware
  → AuthMiddleware               (detects not authenticated)
    ↳ returns 302 → /auth        (short-circuits, controller never runs)
Response ←─────────────────────────────────────
/auth/* routes are outside the group, so AuthMiddleware never runs for them - unauthenticated users can always reach the login page.
